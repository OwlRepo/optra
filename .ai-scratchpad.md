## Context

**What's in scope — decorative content icons, six instances across four files:**
1. `apps/web/app/page.tsx:146` — the "Live answer preview" mockup card's message icon
2. `apps/web/app/page.tsx:184` — the "Trusted workflow" pillars section's per-card icon
3. `apps/web/app/page.tsx:204` — the "Product overview" features section's per-card icon
4. `apps/web/app/workspaces/[id]/page.tsx:190` — the Overview page's quick-link card icons
5. `apps/web/app/workspaces/[id]/page.tsx:213` — the Activity feed's per-event-type icon
6. `apps/web/app/workspaces/[id]/chat/page.tsx:571` — the chat page's "Sources" panel header icon

**What's explicitly out of scope — do not touch:** the app's actual brand/logo mark (`<Sparkles>` in a colored square, used consistently as the "Second Brain" wordmark's icon on the landing page header, both `loading.tsx` skeletons, and all three auth pages' `sidebarHeader`-equivalent brand blocks). That's the product's visual identity, not decorative filler — `DESIGN.md`'s decision targets icon-as-content-decoration, not the logo. Also out of scope: the workspace-name initial-letter avatar circles in `AppShell`/`WorkspaceNav`'s `sidebarHeader` (e.g. `{workspace?.name?.[0]?.toUpperCase()}`) — those are identity avatars (same pattern as any app's user/workspace avatar), a different and legitimate use of a colored circle, not the AI-slop pattern being removed.

**The fix, same shape in all six places:** remove the colored circular/rounded wrapper (`<div>`/`<span>` with `rounded-2xl bg-{primary,accent,secondary}/N ... size-{10,11,12}`), keep the icon itself, rendered bare. Where the wrapper carried a semantic color (`text-primary`, `text-accent-foreground`), move that color directly onto the bare icon so the existing color-coding (e.g. accent for quick-links vs primary for the hero mockup) isn't lost — only the circle/padding/background goes away, not the color meaning. Icon size stays `size-5` (already the icon's own size in every instance — only the wrapper's larger `size-10/11/12` box goes away).
