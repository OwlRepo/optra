## Feature: Web documentation scraping → ingest (production-grade) — COMPLETE handoff (W1–W4)

Add a "Web source" to a knowledge base: crawl an in-domain docs site, extract clean article text, and feed each page through the EXISTING ingest pipeline so support agents can query it. Demo target: `https://support.supremainc.com/en/support/home`.

Implement the four slices IN ORDER (W1 → W2 → W3 → W4). Each slice is independently runnable + verifiable; run that slice's verification before moving on. Strict TDD throughout (failing test first).

Design decisions (recorded, do not revert):
- **Custom crawler, NOT LangChain `RecursiveUrlLoader`.** Its options (`excludeDirs/extractor/maxDepth/timeout/preventOutside/callerOptions`) cannot set a User-Agent, honor robots.txt, or throttle per-request — all required for production politeness. We build a small bounded BFS crawler reusing installed `cheerio` (links), `jsdom` (DOM), `p-limit` (concurrency) + new deps `@mozilla/readability` (main-content extraction) and `robots-parser`.
- **Maximal reuse:** a scraped page → clean text stored as `.txt` in SeaweedFS → `documents` row (`sourceUrl`=page URL, `storageKey`=.txt) → the EXISTING `IngestProcessor` (`loadText → chunkDocument → embedChunks → syncChunks`) runs UNCHANGED. One document per page = source attribution; re-crawl upserts by `sourceUrl` and chunk-level SHA-256 dedup avoids re-embedding unchanged content.

**`packages/ai/src/web/crawl.ts` (NEW):**
```ts
export interface CrawlOptions {
  maxDepth?: number          // default 3, clamp [0,5]
  maxPages?: number          // default 500, clamp [1,2000]
  includePrefixes?: string[] // only follow URLs whose pathname startsWith one of these
  excludeDirs?: string[]     // skip URLs whose pathname includes any of these
  concurrency?: number       // default 3, clamp [1,5]
  requestDelayMs?: number    // default 500; effective delay = max(this, robotsCrawlDelay)
  timeoutMs?: number         // default 20000
  userAgent?: string         // default 'MnemraBot/1.0 (+https://mnemra.com/bot)'
  respectRobots?: boolean    // default true
  fetchImpl?: typeof fetch   // injectable for tests; default global fetch
}
export interface CrawledPage { url: string; title: string; content: string }
export async function crawlSite(seedUrl: string, options?: CrawlOptions): Promise<CrawledPage[]>
```
Behavior (bounded BFS): parse seed → origin/host (reject non-http(s)); if `respectRobots` fetch+parse `${origin}/robots.txt` (UA+timeout; on failure default-allow + log); BFS queue `{url,depth}` from canonical seed (depth 0), `visited:Set` keyed on canonical URL, stop when queue empty OR `results.length>=maxPages`; process frontier with `p-limit(concurrency)`, wait `max(requestDelayMs, robotsCrawlDelay)` between requests; per URL skip if visited / robots-disallowed / not same-origin / fails `includePrefixes` / matches `excludeDirs`, then mark visited; fetch with `{headers:{'User-Agent':userAgent, Accept:'text/html'}, signal:AbortSignal.timeout(timeoutMs)}`, retry ≤2 on network/5xx; skip non-2xx + non-`text/html`; extract via Readability (`new JSDOM(html,{url})` → `new Readability(dom.window.document).parse()` → `textContent`+`title`, collapse whitespace), fallback cheerio `body` text + `<title>`; skip near-empty (<50 chars); if `depth<maxDepth` extract `a[href]` via cheerio, resolve `new URL(href,pageUrl)`, canonicalize, enqueue in-scope unvisited at `depth+1`; push `{url:canonical,title,content}`; catch per-page errors (skip, never abort).
Exported helpers (for tests): `canonicalizeUrl(raw, base?)` (resolve relative, lowercase host, drop hash, drop default ports, remove `utm_*`/`gclid`/`fbclid`/`mc_eid`, strip trailing slash except root), `isInScope(url,{origin,includePrefixes,excludeDirs})`, `extractContent(html,url)`, `extractLinks(html,pageUrl)`.

**`packages/ai/src/index.ts` (MODIFY):** add `export * from './web/crawl'`.

### Tests — `packages/ai/src/web/crawl.spec.ts` (inject `fetchImpl` mock; fake in-memory site map incl. `robots.txt`)
1. `canonicalizeUrl` collapses `/home`, `/home/`, `/home#a`, `/home?utm_source=x` → one URL.
2. same-origin + `includePrefixes` scope: off-host + out-of-prefix (`/blog`) not crawled.
3. `maxDepth`: depth-2 reaches articles; depth-1 doesn't.
4. `maxPages` caps result count.
5. dedup: page linked from 2 parents fetched once (assert fetch count).
6. robots: `Disallow`-matched path skipped; `respectRobots:false` ignores.
7. every fetch carries configured `User-Agent`.
8. Readability excludes nav/sidebar/footer boilerplate, keeps article text; `title` correct.
9. resilience: a 500 (post-retry) / non-HTML page is skipped, crawl still returns others.

  2. `try`: `const pages = await crawlSite(url, { maxDepth, maxPages, includePrefixes, concurrency:3, requestDelayMs:500, timeoutMs:20000, userAgent: config UA or default, respectRobots:true })`. `update scrapeRuns set pagesFound: pages.length`.

## Risk Register Notes

Deep. Crawl politeness + bounds enforced in W1 (robots, UA, concurrency/delay/timeout/retries, depth/page clamps, same-origin + path scope, per-page error isolation). Multi-tenant: scrape runs nested under `:workspaceId` (member guard) + per-workspace document quota (`MAX_DOCS_PER_WORKSPACE`) prevents one tenant exhausting DB/embedding budget; `maxPages` clamped to remaining quota. Idempotent: re-scrape upserts by `sourceUrl`; chunk-level hash-diff avoids duplicate embeddings. Content extraction (Readability) keeps boilerplate out of embeddings. Crawl-level failures set run `failed` (visible) and never crash the worker. Operational: only crawl sites the customer is entitled to index (target-site ToS) — flagged to the user.
